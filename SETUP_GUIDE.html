STEP 1
🔥 Create Firebase Project
Firebase is the backend. It handles login, database, and file storage — all for free.

1.1 — Create the project

  1. Go to console.firebase.google.com
  2. Click "Add project"
  3. Enter name: campusconnect
  4. Disable Google Analytics (optional)
  5. Click "Create project"

1.2 — Register your web app

  1. In the Firebase console, click the </> (Web) icon
  2. App nickname: CampusConnect Web
  3. Click "Register app"
  4. You'll see a code block with firebaseConfig — copy it
STEP 2
⚙️ Paste Your Firebase Config
Open js/firebase-config.js and replace the placeholder config with your actual values.
⚠️ Important: Each Firebase project has unique keys. Never use someone else's config — your data will go into their project.
// js/firebase-config.js
// ✅ REPLACE THIS WITH YOUR ACTUAL FIREBASE CONFIG

import { initializeApp } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js';
import { getAuth } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js';
import { getFirestore } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js';
import { getStorage } from 'https://www.gstatic.com/firebasejs/10.12.0/firebase-storage.js';

const firebaseConfig = {
  apiKey: "AIzaSy...",              // ← Your key here
  authDomain: "your-app.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-app.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

const app = initializeApp(firebaseConfig);
export const auth = getAuth(app);
export const db = getFirestore(app);
export const storage = getStorage(app);
STEP 3
🔐 Enable Authentication
Enable sign-in methods so users can register and login.
  1. In Firebase Console → go to Authentication (left sidebar)
  2. Click "Get started"
  3. Click Email/Password
  4. Toggle Enable → Save
ℹ️ Email/Password is the primary sign-in method. The OTP verification shown during registration is Firebase's built-in email verification (link sent to email, not SMS). If you need SMS OTP, you'd need to enable Phone auth and pay for Firebase's SMS service.
STEP 4
🗄️ Set Up Firestore Database
Firestore stores all your app data: users, posts, doubts, tests, etc.

4.1 — Create the database

  1. In Firebase Console → Firestore Database
  2. Click "Create database"
  3. Select "Start in test mode" (you'll secure it later)
  4. Choose a server location close to India (e.g., asia-south1 (Mumbai))
  5. Click "Enable"

4.2 — Set Firestore Security Rules

Go to Firestore → Rules tab → Replace with:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Users: anyone authenticated can read, only the user can write their own doc match /users/{userId} { allow read: if request.auth != null; STEP 5
📦 Set Up Firebase Storage
Storage holds all uploaded files (PDFs, notes, resources).
  1. In Firebase Console → Storage
  2. Click "Get started"
  3. Start in test mode → Click "Next" → "Done"

Storage Security Rules

Go to Storage → Rules tab → Replace with:

rules_version = '2'; service firebase.storage { match /b/{bucket}/o { // Allow authenticated users to read all files match /{allPaths=**} { allow read: if request.auth != null; } // Allow users to upload to their own folder (max 50MB) match /resources/{userId}/{fileName} { allow write: if request.auth != null && request.auth.uid == userId && request.resource.size < 50 * 1024 * 1024 && request.resource.contentType.matches('application/.*|text/.*'); } } }
STEP 6

Indexes are needed for multi-field queries. Create these in Firestore → Indexes → Composite.
⚠️ If you see a Firestore error about missing indexes in the browser console, click the link in the error — it will auto-create the index for you.

STEP 7
👑 Create the First Admin Account
You need to manually create the admin user since admin registration is not open to public.

Method 1 — Register then promote (easiest)

  1. First, add your college's admin ID to Firestore manually:
    • Go to Firestore → registeredIds collection
    • Add a document with fields: id: "ADMIN001", type: "student", used: false
  2. Open index.html and register with that ID, choosing role "Admin"
  3. After registration, go to Firestore → users collection
  4. Find your user document → Edit → change role to admin, approved to true
  5. Login again — you'll be redirected to admin.html ✅
💡 Once you have admin access, use the Admin Dashboard → "Registered IDs" tab to add all student and faculty IDs in bulk. Students will only be able to register using IDs you've added.
STEP 8
🚀 Deploy Your App
Host CampusConnect so students can access it from any device. Several free options:

Option A — Firebase Hosting (Recommended)

  1. Install Node.js from nodejs.org
  2. Open terminal in your campusconnect/ folder
  3. Run these commands: 
    npm install -g firebase-tools
firebase login
firebase init hosting
# When asked for public directory: type  .  (a single dot, meaning current folder)
# Configure as single-page app? No
# Set up automatic builds? No
firebase deploy
  4. Your app goes live at: https://your-project-id.web.app

Option B — Netlify (Drag & Drop, easiest)

  1. Go to netlify.com → Sign up free
  2. Drag your entire campusconnect/ folder onto the Netlify dashboard
  3. It's live instantly at a random URL
  4. You can set a custom domain later

Option C — GitHub Pages

  1. Create a GitHub account → Create a new repository
  2. Upload all files → Go to Settings → Pages → Deploy from main branch
  3. Live at: https://yourusername.github.io/campusconnect
STEP 9
🌐 Add Authorized Domains
After deploying, add your domain to Firebase so login works from that URL.
  1. Firebase Console → Authentication → Settings tab
  2. Scroll to "Authorized domains"
  3. Click "Add domain"
  4. Add your deployment URL (e.g., your-app.web.app or your custom domain)
⚠️ If login fails after deployment with an "unauthorized domain" error, this is the fix.

HELP
🔧 Common Issues & Fixes

❌ "FirebaseError: Missing or insufficient permissions"

→ Your Firestore rules are blocking access. Make sure you pasted the correct rules in Step 4. Wait 30 seconds for rules to apply.

❌ Login works locally but fails after deployment

→ Add your deployment domain to Firebase Authentication → Settings → Authorized domains (Step 9).

❌ "The query requires an index" error in console

→ Click the link in the console error — it auto-creates the index. Wait 2-3 minutes for index to build.

❌ File upload fails

→ Check Storage rules in Step 5. Make sure Storage is enabled and rules are saved.

❌ Users stuck "Pending" — can't see the platform

→ Admin must approve them in Admin Dashboard → Pending Users. This is a feature, not a bug.

❌ Email verification not arriving

→ Check spam folder. Firebase sends from noreply@your-project.firebaseapp.com. You can customize the email template in Auth → Templates.

❌ App works locally but shows blank page on deploy

→ Make sure all file paths use relative paths (../css/main.css) not absolute paths. Check browser console for 404 errors.
